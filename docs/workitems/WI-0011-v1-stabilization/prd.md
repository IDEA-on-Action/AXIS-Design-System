# PRD — v1.0 안정화

> 작성일: 2026-02-01 | 작성자: AXIS Team | 상태: Draft

---

## 1. 개요

### 1.1 배경
- 현재 v0.7.0으로 Phase 0~7 전체 완료 (53/53 항목, 100%)
- 6개 패키지(`tokens`, `ui-react`, `agentic-ui`, `theme`, `cli`, `mcp`) 구현 완료
- v1.0 정식 릴리스를 위해 품질 보증, 문서 정비, 배포 파이프라인 검증이 필요

### 1.2 목표
- 목표 1: 전체 패키지 테스트 커버리지 확보 (주요 경로 80%+)
- 목표 2: 문서 최종 검토 및 정비 (API 문서, 가이드, 예제)
- 목표 3: npm 정식 배포 파이프라인 검증 및 실행
- 성공 지표: v1.0.0 npm 배포 완료, 문서 사이트 정식 공개

### 1.3 범위
- 포함: 테스트 작성, 문서 정비, npm 배포, CHANGELOG 작성, 버전 범핑
- 제외 (Non-goals): 새 컴포넌트 추가, 새 기능 개발, 디자인 변경

---

## 2. 사용자 및 사용 사례

### 2.1 대상 사용자
- 개발자: AXIS DS를 프로젝트에 도입하려는 React 개발자
- 디자이너: 디자인 토큰 및 컴포넌트 참조 문서 활용

### 2.2 핵심 사용 시나리오
1. 개발자가 `npm install @axis-ds/ui-react`로 안정적인 v1.0 패키지를 설치
2. 문서 사이트에서 컴포넌트 API, 사용 예제, 가이드를 참조

---

## 3. 요구사항

### 3.1 기능 요구사항
- FR1: 주요 패키지(`ui-react`, `agentic-ui`, `cli`)에 단위 테스트 추가
- FR2: 문서 사이트 전체 페이지 검수 (깨진 링크, 누락 콘텐츠 확인)
- FR3: npm 배포 스크립트 검증 (dry-run → 실배포)
- FR4: CHANGELOG.md v1.0.0 섹션 작성
- FR5: 패키지 버전 1.0.0으로 범핑

### 3.2 비기능 요구사항
- 접근성: 기존 WCAG 2.1 AA 준수 상태 유지
- 성능: 번들 사이즈 현황 문서화
- 호환성: React 19, Next.js 15 호환 확인

## 6. 수용 기준 (Acceptance Criteria)

- AC1: `pnpm test` 실행 시 주요 패키지 테스트 전체 통과
- AC2: `pnpm build` 전 패키지 빌드 성공, 타입 체크 통과
- AC3: 문서 사이트 전체 페이지 접근 가능, 깨진 링크 0건
- AC4: npm dry-run 배포가 오류 없이 완료
- AC5: CHANGELOG.md에 v1.0.0 섹션이 작성되고 모든 변경사항 포함
- AC6: 전 패키지 package.json 버전이 1.0.0

---

## 7. 가정 (Assumptions)

- A1: 기존 구현된 컴포넌트에 breaking change가 필요하지 않음
- A2: npm 레지스트리 접근 권한이 확보된 상태
- A3: 현재 CI/CD 파이프라인이 정상 동작 중

---

## 8. 리스크

| 리스크 | 영향 | 완화 방안 |
|--------|------|-----------|
| 테스트 작성 시 기존 버그 발견 | 릴리스 지연 | 크리티컬만 수정, 나머지는 v1.0.1로 미룸 |
| npm 배포 권한 문제 | 배포 불가 | 사전에 권한 확인 및 테스트 |
| 문서 누락/오류 다수 발견 | 작업량 증가 | 핵심 페이지 우선 정비 |
