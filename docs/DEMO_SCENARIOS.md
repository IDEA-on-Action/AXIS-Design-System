## 시나리오 1: 세미나 → Signal → Scorecard

### 📋 개요

**스토리**: BD팀원이 외부 기술 세미나에 참석 후 사업기회 신호를 포착하여 평가하는 과정

**소요 시간**: 5분
**사용 워크플로**: WF-01 (Seminar Pipeline), WF-02 (Interview-to-Brief)

### 🎬 데모 스크립트

#### Step 1: 세미나 Activity 등록 (1분)

**화면**: `/seminar` 페이지

```
[내레이션]
"BD팀 김대리가 'AI 기반 스마트팩토리' 세미나에 참석했습니다.
세미나에서 들은 내용을 시스템에 등록합니다."
```

**입력 데이터**:
```json
{
  "title": "AI 기반 스마트팩토리 혁신 세미나",
  "date": "2026-01-15",
  "organizer": "한국산업기술협회",
  "attendee": "김대리",
  "summary": "제조업 AI 도입 현황 및 KT의 기회 영역 발표. 특히 중소제조업체의 품질검사 자동화 수요가 급증하고 있으며, 현재 시장에 적합한 솔루션이 부재한 상황.",
  "key_insights": [
    "중소제조업 품질검사 자동화 수요 급증",
    "기존 솔루션은 대기업 중심으로 가격 장벽 높음",
    "KT 5G + AI 조합으로 경쟁력 있는 솔루션 가능"
  ],
  "potential_contacts": [
    {"name": "박상무", "company": "한국산업기술협회", "role": "스마트팩토리 TF장"}
  ]
}
```

**예상 결과**:
- Activity ID 생성
- Confluence Activity Log 페이지 자동 생성

### ✅ 시나리오 1 핵심 포인트

1. **자동화**: 세미나 등록 → Signal 추출 → Scorecard 평가 자동 파이프라인

3. **추적성**: 모든 활동이 Activity → Signal → Scorecard로 연결됨

## 시나리오 2: VoC 분석 → Brief → Confluence 동기화

### 📋 개요

**스토리**: 고객 VOC 데이터를 분석하여 사업기회를 발굴하고 Brief를 자동 생성하는 과정

**소요 시간**: 5분
**사용 워크플로**: WF-03 (VoC Mining), WF-02 (Brief 생성), WF-06 (Confluence Sync)

### 🎬 데모 스크립트

#### Step 1: VoC 데이터 업로드 (1분)

**화면**: `/inbox` 페이지 또는 API 직접 호출

```
[내레이션]
"영업팀에서 수집한 최근 한 달간 고객 피드백 데이터가 있습니다.
이 데이터를 VoC Mining 파이프라인에 입력합니다."
```

**입력 데이터** (CSV 형식):
```csv
date,customer,feedback,category

2026-01-11,B물류,물류센터 자동화 관심. 인건비 상승으로 무인화 검토 중,물류자동화
2026-01-12,C제조,스마트팩토리 구축 희망. 정부 지원사업 연계 가능 여부 문의,스마트팩토리
2026-01-13,D유통,재고 관리 AI 솔루션 필요. 현재 엑셀로 관리하여 오류 빈번,재고관리
2026-01-14,E제조,품질검사 자동화 시급. 인력 부족으로 검사 품질 저하,품질관리
```

**API 호출**: `POST /api/workflows/voc-mining`
