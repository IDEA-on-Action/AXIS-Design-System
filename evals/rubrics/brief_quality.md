# Brief 품질 평가 루브릭

> LLM-as-Judge 채점기용 루브릭

## 평가 목적

이 루브릭은 AI 에이전트가 생성한 1-Page Opportunity Brief의 품질을 평가합니다.
각 차원별로 1-5점 척도로 평가하며, 점수별 기준을 명확히 정의합니다.

---

## 1. Problem Clarity (문제 명확성)

**평가 질문**: Pain Point가 명확하고 정량적으로 기술되었는가?

| 점수 | 기준 |
|------|------|
| 5 | Pain Point가 구체적이고 정량 데이터(%, 금액, 시간 등)로 뒷받침됨. 비즈니스 영향이 명확히 수치화됨. |
| 4 | Pain Point가 명확하고 일부 정량 데이터 포함. 비즈니스 영향 언급됨. |
| 3 | Pain Point가 기술되었으나 정량 데이터 부족. 영향도 모호함. |
| 2 | Pain Point가 불명확하거나 지나치게 일반적임. |
| 1 | Pain Point가 기술되지 않았거나 완전히 부정확함. |

**핵심 체크포인트**:
- [ ] 고객의 Pain Point가 명시됨
- [ ] 정량적 데이터 포함 (%, 금액, 시간, 건수 등)
- [ ] 비즈니스 영향 (비용, 매출, 효율성)이 연결됨

---

## 2. Solution Fit (솔루션 적합성)

**평가 질문**: 제안 솔루션이 Pain Point를 적절히 해결하는가?

| 점수 | 기준 |
|------|------|
| 5 | 솔루션이 Pain Point와 직접 연결되고, 구체적인 해결 방안 제시. 기술적 타당성 높음. |
| 4 | 솔루션이 Pain Point를 해결하며, 대략적인 접근 방식 설명됨. |
| 3 | 솔루션이 언급되었으나 Pain Point와의 연결이 약함. |
| 2 | 솔루션이 불명확하거나 Pain Point와 맞지 않음. |
| 1 | 솔루션이 제시되지 않거나 완전히 부적합함. |

**핵심 체크포인트**:
- [ ] 솔루션이 Pain Point에 직접 대응
- [ ] 기술적 접근 방식이 명시됨
- [ ] 실현 가능성이 있음

---

## 3. Business Impact (비즈니스 영향)

**평가 질문**: 예상 ROI/효과가 설득력 있게 제시되었는가?

| 점수 | 기준 |
|------|------|
| 5 | ROI가 정량적으로 계산됨 (예: "연간 10억원 절감", "효율 30% 향상"). 근거 명확. |
| 4 | 비즈니스 영향이 수치로 제시되나 근거가 다소 약함. |
| 3 | 비즈니스 영향이 정성적으로만 언급됨 (예: "효율 향상 기대"). |
| 2 | 비즈니스 영향이 모호하거나 과장됨. |
| 1 | 비즈니스 영향이 언급되지 않음. |

**핵심 체크포인트**:
- [ ] 예상 효과가 정량화됨
- [ ] ROI 또는 비용 절감 금액 제시
- [ ] 효과 산출 근거가 논리적

---

## 4. Actionability (실행 가능성)

**평가 질문**: Next Steps가 구체적이고 실행 가능한가?

| 점수 | 기준 |
|------|------|
| 5 | Next Steps가 구체적 (담당자, 일정, 산출물 명시). 즉시 실행 가능. |
| 4 | Next Steps가 명확하나 일부 세부사항 부족. |
| 3 | Next Steps가 있으나 추상적 (예: "추가 논의 필요"). |
| 2 | Next Steps가 불명확하거나 실행 불가능. |
| 1 | Next Steps가 없음. |

**핵심 체크포인트**:
- [ ] 구체적인 Action Item 제시
- [ ] 담당자/일정 언급
- [ ] 다음 단계가 명확

---

## 5. Completeness (완성도) - 선택적

**평가 질문**: Brief가 필수 섹션을 모두 포함하고 일관성 있는가?

| 점수 | 기준 |
|------|------|
| 5 | 모든 필수 섹션 포함, 일관된 톤, 오탈자 없음, 적절한 분량. |
| 4 | 필수 섹션 포함, 경미한 일관성 이슈. |
| 3 | 일부 섹션 누락 또는 내용 불균형. |
| 2 | 여러 섹션 누락, 일관성 부족. |
| 1 | 대부분의 섹션 누락 또는 형식 미준수. |

**필수 섹션**:
- [ ] Problem Statement / Pain Point
- [ ] Proposed Solution
- [ ] Business Impact / ROI
- [ ] Next Steps / Action Items

---

## 평가 시 주의사항

1. **Outcome 중심**: 에이전트가 "어떻게" 도달했는지보다 "무엇을" 만들었는지 평가
2. **부분 점수**: 이진(pass/fail)보다 연속적 점수로 개선 신호 제공
3. **Unknown 허용**: 정보 부족 시 "평가 불가" 가능 (환각 방지)
4. **근거 명시**: 각 점수에 대한 이유를 transcript/output 기반으로 설명

---

## 총점 계산

```
총점 = (problem_clarity × 0.25) + (solution_fit × 0.25) +
       (business_impact × 0.25) + (actionability × 0.25)

# 5점 만점 → 100점 환산
최종점수 = 총점 × 20
```

## 판정 기준

| 점수 범위 | 판정 |
|-----------|------|
| 80-100 | PASS (우수) |
| 60-79 | MARGINAL (개선 필요) |
| 0-59 | FAIL (재작성 필요) |
