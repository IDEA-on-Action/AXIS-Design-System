# 기능 구현 계획서

## 목표

{{FEATURE_NAME}} 기능을 체계적으로 구현하기 위한 계획을 수립합니다.

## 입력 정보

| 변수 | 설명 | 필수 | 예시 |
|------|------|------|------|
| `{{FEATURE_NAME}}` | 구현할 기능명 | Y | 다크모드 지원 |
| `{{TARGET_COMPONENT}}` | 대상 컴포넌트/모듈 | Y | ThemeProvider |
| `{{PRIORITY}}` | 우선순위 | N | P1 |
| `{{DEADLINE}}` | 목표 완료일 | N | 2024-12-31 |

## 수행 단계

### 단계 1: 요구사항 분석

1. 기능 요구사항 정의
2. 비기능 요구사항 식별
3. 제약 조건 파악
4. 의존성 분석

### 단계 2: 영향 범위 분석

1. 수정 대상 파일 목록 작성
2. 영향받는 컴포넌트 식별
3. Breaking changes 확인
4. 테스트 범위 결정

### 단계 3: 설계

1. 아키텍처 설계
2. 인터페이스 정의
3. 데이터 흐름 설계
4. 에러 처리 전략

### 단계 4: 구현 계획

1. 작업 분해 (WBS)
2. 우선순위 결정
3. 마일스톤 정의
4. 리스크 식별
