# /ax:health-check Command (프로젝트 점검)

프로젝트의 전반적인 상태를 점검합니다.

## 사용법

```
/ax:health-check [--quick] [--full] [--fix] [--json]
```

## 인자

| 인자 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `--quick` | | 빠른 점검 (타입/린트만) | |
| `--full` | | 전체 점검 (테스트 포함) | |
| `--fix` | | 자동 수정 가능한 항목 수정 | |
| `--json` | | JSON 형식으로 결과 출력 | |

## 실행 워크플로

```
1. 프로젝트 의존성 및 구조 확인
2. 타입 체크 실행 (mypy, tsc)
3. 린트 검사 실행 (ruff, ESLint)
4. 빌드 테스트
5. 버전 동기화 확인
6. 점검 결과 리포트
```

## 점검 항목

| 항목 | Python | TypeScript |
|------|--------|------------|
| 타입 체크 | mypy | tsc |

| 빌드 | pip install -e . | pnpm build |

## 에러 처리

| 에러 | 메시지 | 해결 방법 |
|------|--------|----------|
| 타입 에러 | "mypy/tsc 에러 발견" | 에러 수정 후 재실행 |
| 린트 에러 | "ruff/ESLint 에러 발견" | `--fix` 옵션으로 자동 수정 |
| 빌드 실패 | "빌드 중 오류 발생" | 에러 메시지 확인 |
| 버전 불일치 | "버전 동기화 필요" | 버전 통일 후 태그 생성 |
